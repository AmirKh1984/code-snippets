1.1 - A Brief Explanation

The idea of browser-based exercise files

This project grew out of a need to provide a consistent code snippet experience for viewers. The idea is to provide a web-based set of code snippets that can be styled to work in all browsers and is easily read and accessed. As a bonus the code snippet document itself can serve as a best-practice example of a simple web-based application.

The baseline code snippet document uploaded as the initial commit is currently shipping with a series of WordPress courses, debuting with WordPress: Building Themes from Scratch Using Underscores.

1.2 - Goals for the project

The project was built with a few basic goals in mind:

  • HTML/CSS/JS-based code snippets
  • Code highlighting
  • Snippet navigation
  • Responsive Web Design

Moving forward some additional goals (to be expanded) are:

  • Persistent nav menu at the top
  • “Click to copy” button in snippets
  • Paged navigation
  • Gist and/or Codepen integration for continuous updates

1.3 - Components used in this project

For styling this project uses two external resources: Normalize.css and type sizing from by Jason Pamental. The former is found in parts/normalize.css, the latter is baked into parts/style.css.

For code highlighting Prism.js is used through parts/prism.js and parts/prism.css. Prism was chosen because the styling is entirely CSS based and applied with simple classes while the markup uses only 

<pre><code>
to highlight code which is correct to web standards.

The Prism.js website is unjustifiably short on code examples but provides some glimpses of the impressive potential this extension has including line highlighting and other fun stuff. Identifying a particular block of code for highlighting is done by applying a simple semantic class as in the example below. More examples are added to Chapter 2 of this document. Note the use of HTML entities for the opening less-than sign in the code example itself.

Prism.js markup example for HTML:


<pre><code class="language-markup">
&lt;h1>The Title&lt;/h1>
&lt;p>This is where the paragraph text will go.&lt;/p>
</code></pre>

If you find any issues or want to propose new features, do so in the Issues tab of the original repo.

2.4 - PHP

In a .php file:


function my_simone_paging_nav() {
    // Don't print empty markup if there's only one page.
    if ( $GLOBALS['wp_query']->max_num_pages < 2 ) {
        return;
    }

    $paged        = get_query_var( 'paged' ) ? intval( get_query_var( 'paged' ) ) : 1;
    $pagenum_link = html_entity_decode( get_pagenum_link() );
    $query_args   = array();
    $url_parts    = explode( '?', $pagenum_link );

    if ( isset( $url_parts[1] ) ) {
        wp_parse_str( $url_parts[1], $query_args );
    }

    $pagenum_link = remove_query_arg( array_keys( $query_args ), $pagenum_link );
    $pagenum_link = trailingslashit( $pagenum_link ) . '%_%';

    $format  = $GLOBALS['wp_rewrite']->using_index_permalinks() && ! strpos( $pagenum_link, 'index.php' ) ? 'index.php/' : '';
    $format .= $GLOBALS['wp_rewrite']->using_permalinks() ? user_trailingslashit( 'page/%#%', 'paged' ) : '?paged=%#%';

    // Set up paginated links.
    $links = paginate_links( array(
        'base'     => $pagenum_link,
        'format'   => $format,
        'total'    => $GLOBALS['wp_query']->max_num_pages,
        'current'  => $paged,
        'mid_size' => 2,
        'add_args' => array_map( 'urlencode', $query_args ),
        'prev_text' => __( '← Previous', 'my-simone' ),
        'next_text' => __( 'Next →', 'my-simone' ),
        'type'      => 'list',
    ) );

    if ( $links ) :

    ?>
    <nav class="navigation paging-navigation" role="navigation">
        <h1 class="screen-reader-text"><?php _e( 'Posts navigation', 'my-simone' ); ?></h1>
            <?php echo $links; ?>
    </nav><!-- .navigation -->
    <?php
    endif;
}
